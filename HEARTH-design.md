## Problem Statement

Hearth is a three-sided AI-powered food security platform connecting seekers (food-insecure students/community members), providers (home cooks, restaurants, food banks), and sponsors (local government, corporate) — built for the Anthropic Social Impact Hackathon. The UI must eliminate charity stigma through Airbnb-caliber design: food-first cards, warm photography, no handout language anywhere.

## What Makes This Cool

The platform uses Claude as the operating system — not a chatbot bolted on. Claude onboards, parses, matches, calls, cooks, and reports. The live Twilio voice call during the demo is the moment. The notification Claude writes ("the jollof rice you made tonight went to a student who hadn't had food that felt like home in weeks") is the emotional peak.

## Airtable Schema (James creates these tables in Hour 1)

### food_listings
| Field | Type |
|---|---|
| id | Auto |
| listing_title | Text |
| listing_emoji | Text |
| food_items | Long text |
| portions | Number |
| dietary_flags | Multi-select: halal, vegan, vegetarian, gluten-free, kosher, nut-free, dairy-free |

| pickup_location | Text |
| pickup_window | Text |
| prep_date | Text |
| safety_passed | Checkbox |
| status | Select: available, reserved, completed |
| points_awarded | Number (default: 15) |
| created_at | Date |

### seeker_profiles
| Field | Type |
|---|---|
| session_id | Text (UUID from localStorage) |
| dietary_restrictions | Multi-select |
| health_conditions | Multi-select: diabetic, hypertension, pregnancy |
| cultural_preferences | Long text |
| kitchen | Select: microwave, hotplate, full |
| favorite_ingredients | Long text |
| avoid | Long text |
| created_at | Date |

### providers
| Field | Type |
|---|---|
| id | Auto |
| name | Text |
| contact | Phone |
| hearth_hearts | Number (start: 0) |
| total_portions | Number |
| badge | Select: Feeder (0-99), Hero (100-299), Legend (300+) |
| joined | Date |

### matches
| Field | Type |
|---|---|
| id | Auto |
| seeker_session | Text |
| listing_id | Text |
| status | Select: matched, confirmed, completed |
| notification_sent | Checkbox |
| created_at | Date |

**Hearth Hearts rules**: +15 per successful match. Badge upgrades: Feeder at 0, Hero at 100, Legend at 300. PointsTracker shows current total + distance to next milestone.

## Minimal Fetch.ai Agent Scope

One agent. One job: receive a match result + provider contact, kick off the call flow asynchronously.

```python
# agents/hearth_agent.py — ~40 lines total
# Input: { listing_id, seeker_session, provider_phone, food_items, pickup_window }
# Action: POST to /api/twilio/call
# Output: { call_script, status }
# That's it. No complex orchestration.
```

Don't wire Fetch.ai into the demo flow as a required dependency. Show it as "this is how we'd scale the async pipeline." The demo works without it if it breaks.
