## What We're Building

Three-sided AI-powered food security platform:
- **Seekers** (students, food-insecure individuals) — chat to find food
- **Providers** (home cooks, restaurants, neighbors) — drop a message to share food
- **Sponsors** (local government, corporate) — see impact data

Claude IS the product. It onboards, parses, matches, generates call scripts, writes notifications, cooks recipes, and produces reports.

**Stack:** Next.js 14 + Tailwind + Claude API (claude-sonnet-4-20250514) + Airtable + minimal Fetch.ai

## lib/claude.ts (Arshia owns — James imports from this)
```typescript
import Anthropic from "@anthropic-ai/sdk";

export const anthropic = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY,
});

// Use for streaming endpoints (onboard)
export function createStream(model: string, system: string, messages: any[]) {
  return anthropic.messages.stream({
    model,
    max_tokens: 1024,
    system,
    messages,
  });
}

// Use for JSON endpoints (parse-listing, match, notify, recipe, report)
export async function createMessage(model: string, system: string, userContent: string) {
  const response = await anthropic.messages.create({
    model,
    max_tokens: 2048,
    system,
    messages: [{ role: "user", content: userContent }],
  });
  return response.content[0].text;
}
```

## Error Handling Pattern (James applies to parse-listing and match routes)

Claude occasionally returns text that isn't valid JSON. Wrap all JSON routes with this pattern:

```typescript
// app/api/claude/parse-listing/route.ts
export async function POST(req: Request) {
  try {
    const { message } = await req.json();
    const rawText = await createMessage(MODEL, PARSE_SYSTEM, message);

    // Extract JSON from Claude's response (may be wrapped in ```json ... ```)
    const jsonMatch = rawText.match(/```json\n?([\s\S]*?)```/) || rawText.match(/(\{[\s\S]*\})/);
    if (!jsonMatch) {
      return Response.json({ error: 'parse_failed', raw: rawText }, { status: 422 });
    }

    const listing = JSON.parse(jsonMatch[1]);
    return Response.json(listing);
  } catch (error) {
    console.error('parse-listing error:', error);
    return Response.json({ error: 'internal_error' }, { status: 500 });
  }
}
```

Apply the same try/catch pattern to: match, notify, recipe, report, twilio/call.

## JSON Streaming Detection (Jason's ChatInterface.tsx)

Claude ends the onboarding with a JSON block fenced in triple backticks. Here's the exact detection pattern:

```typescript
// In ChatInterface.tsx — accumulate the full stream, then detect
const [fullResponse, setFullResponse] = useState('');
const [profile, setProfile] = useState<SeekerProfile | null>(null);

// After stream completes:
useEffect(() => {
  if (!fullResponse) return;
  const jsonMatch = fullResponse.match(/```json\n?([\s\S]*?)```/);
  if (jsonMatch) {
    try {
      const extracted = JSON.parse(jsonMatch[1]) as SeekerProfile;
      setProfile(extracted);
      // Don't show the raw JSON — show "Finding matches..." instead
    } catch (e) {
      console.error('Profile parse failed:', e);
    }
  }
}, [fullResponse]);
```

The JSON block will appear at the END of the stream, after Claude says "Got everything I need. Finding what's available near you now..." — so you can safely wait for the full stream to complete before parsing.

## Twilio Call Strategy (Arshia's /api/twilio/call)

**For the demo: no real call is made.** Claude generates the script. The UI shows it.

```typescript
// app/api/twilio/call/route.ts
const CALL_SCRIPT_PROMPT = `Generate a warm 60-second phone script for Hearth
confirming a food pickup. Context: calling {provider_name} to confirm {food_items}
for pickup at {pickup_time}. The seeker is "a community member."
Introduce as "Hearth, a community food network." Under 90 seconds when spoken.
Natural, warm. Just the words — no stage directions.`;

export async function POST(req: Request) {
  try {
    const { providerName, foodItems, pickupTime } = await req.json();
    const prompt = CALL_SCRIPT_PROMPT
      .replace('{provider_name}', providerName)
      .replace('{food_items}', foodItems)
      .replace('{pickup_time}', pickupTime);

    const script = await createMessage("claude-sonnet-4-20250514", "", prompt);

    return Response.json({
      status: "call_initiated",
      script,
      provider: providerName,
      pickup_time: pickupTime,
    });
  } catch (error) {
    return Response.json({ error: 'call_script_failed' }, { status: 500 });
  }
}
```

**Post-hackathon:** replace the return with `twilio.calls.create({ twiml: ..., to: providerPhone })`. Architecture is already wired.

## Ethics Checklist — Build AND Demo These

- [ ] **Banned words:** "food bank", "charity", "donation", "aid", "welfare", "handout" never appear in seeker-facing Claude outputs. System prompt enforces this.
- [ ] **Safety gate:** `safety_passed: false` blocks listing from Airtable. Provider sees a follow-up question, not an error.
- [ ] **Allergen hard filter:** match prompt EXCLUDES listings where allergen_present matches seeker allergy. Not a soft ranking — a hard exclusion. Demo this: peanut-allergy seeker never sees a nut listing.
- [ ] **Seeker anonymity:** matches table stores session_id only. Provider notification says "a community member" / "a student" — never reveals seeker identity.
- [ ] **Good Samaritan Act:** One-line compliance notice shown to provider before they confirm a listing. Built into ProviderDrop confirmation step.
